## Мета

Опрацювання теми покажчиків у мовах `C` та `C++`, їх призначення, синтаксису, семантики, способів оголошення,
розіменування, адресної арифметики, застосування з одновимірними та двовимірними масивами, використання `void*`,
динамічного виділення пам’яті, а також сучасних практичних підходів до безпечної роботи з пам’яттю.

Основою роботи є контрольні питання з практичної роботи, де розглядаються визначення покажчика, оператори `&` і `*`,
арифметика покажчиків, `void*`, доступ до елементів масивів через покажчики,
функції виділення та звільнення пам’яті, а також відмінності розміщення
одновимірних і двовимірних масивів у пам’яті.

## Базове визначення покажчика

Покажчик це змінна, значенням якої є адреса іншого об’єкта в пам’яті.
Тобто звичайна змінна зберігає саме значення, наприклад число 10,
а покажчик зберігає адресу комірки пам’яті, де це значення розташоване.

> *Наприклад:*
> 
> ```cpp
> int x = 10;
> int* p = &x;
> ```
> 
> *Тут x є звичайною змінною типу `int`, `p` є покажчиком на `int`, а `&x` означає адресу змінної `x`.*

## Основні синтаксичні конструкції покажчиків

| Конструкція    | Назва                          | Суть                                                   | Приклад                   | Особливість                                                               |
|----------------|------------------------|-----------------------------|---------------------------|-----------------------------|
| `T* p`         | Оголошення покажчика           | Оголошує змінну `p`, яка зберігає адресу об’єкта типу `T` | `int* p;`                 | Сам об’єкт ще не створюється, створюється лише покажчик                   |
| `&x`           | Взяття адреси                  | Повертає адресу змінної `x`                            | `p = &x;`                 | Тип результату залежить від типу `x`, для `int x` це `int*`               |
| `*p`           | Розіменування покажчика        | Дає доступ до значення, яке лежить за адресою `p`      | `cout << *p;`             | Якщо `p` невалідний, поведінка невизначена                                |
| `*p = value`   | Запис через покажчик           | Змінює значення об’єкта, на який вказує покажчик       | `*p = 20;`                | Фактично змінюється не покажчик, а об’єкт за адресою                      |
| `p = nullptr`  | Нульовий покажчик              | Покажчик не вказує на жодний об’єкт                    | `int* p = nullptr;`       | У сучасному C++ краще використовувати `nullptr`, а не `NULL`              |
| `p == nullptr` | Перевірка покажчика            | Перевіряє, чи покажчик не вказує на об’єкт             | `if (p == nullptr)`       | Перед розіменуванням покажчик бажано перевіряти                           |
| `p + i`        | Адресна арифметика             | Адреса i-го елемента після `p`                         | `p + 3`                   | Зсув виконується не в байтах, а в елементах типу `T`                      |
| `*(p + i)`     | Доступ через зсув              | Доступ до елемента за зміщенням                        | `*(p + 2)`                | Еквівалентно `p[2]`                                                       |
| `p[i]`         | Індексований доступ через покажчик | Синтаксично зручний доступ до елемента                 | `p[2]`                    | Повністю еквівалентно `*(p + i)`                                          |
| `T** pp`       | Покажчик на покажчик           | Зберігає адресу покажчика                              | `int** pp;`               | Використовується для динамічних двовимірних масивів і зміни покажчика у функції |
| `void* p`      | Узагальнений покажчик          | Може зберігати адресу об’єкта будь якого типу          | `void* p = &x;`           | Перед розіменуванням потрібне приведення типу                             |
| `new T`        | Динамічне створення одного об’єкта | Виділяє пам’ять у купі                                 | `int* p = new int;`       | Звільняється через `delete`                                               |
| `delete p`     | Звільнення одного об’єкта      | Звільняє пам’ять, виділену через `new`                 | `delete p;`               | Не можна застосовувати до масиву                                          |
| `new T[n]`     | Динамічний масив               | Виділяє масив у купі                                   | `int* a = new int[n];`    | Звільняється через `delete[]`                                             |
| `delete[] a`   | Звільнення динамічного масиву  | Звільняє масив, виділений через `new[]`                | `delete[] a;`             | Має відповідати саме `new[]`                                              |
| `malloc(size)` | Виділення пам’яті в C          | Виділяє блок пам’яті в байтах                          | `malloc(n * sizeof(int))` | Не викликає конструктори C++                                              |
| `free(p)`      | Звільнення пам’яті в C         | Звільняє пам’ять, виділену через `malloc`              | `free(p);`                | Не можна змішувати з `new/delete`                                         |

## Положення зірочки при оголошенні

У C++ є кілька стилів запису:

```cpp
int* p;
int *p;
int * p;
```

Усі три записи означають одне й те саме: `p` є покажчиком на `int`.
Важлива особливість виникає при оголошенні кількох змінних:

```cpp
int* a, b;
```

Це не означає, що `a` і `b` обидва покажчики. Тут:

```cpp
a  // int*
b  // int
```

Тому безпечніше писати так:

```cpp
int* a;
int* b;
```

або, якщо в одному рядку:

```cpp
int *a, *b;
```

> Для сучасного `C++` у навчальній і промисловій документації часто використовують стиль `int* p`,
> бо він підкреслює тип: покажчик на `int`. Але з погляду граматики `C++` символ `*` прив’язується до імені змінної,
> а не до типу в цілому.

### Різниця між &, &&, *, **

| Символ   | Контекст   | Значення                                                                                                        |
|----------|------------|--------------------------------|
| `&x`     | Вираз      | Взяти адресу змінної `x`                                                                                        |
| `T& r`   | Оголошення | Посилання на об’єкт типу `T`                                                                                    |
| `a & b`  | Вираз      | Побітове `І`                                                                                                    |
| `T&& r`  | Оголошення | [Rvalue](https://learn.microsoft.com/uk-ua/cpp/cpp/rvalue-reference-declarator-amp-amp?view=msvc-170) посилання |
| `a && b` | Вираз      | Логічне `І`                                                                                                     |
| `*p`     | Вираз      | Розіменувати покажчик                                                                                           |
| `T* p`   | Оголошення | Покажчик на `T`                                                                                                 |
| `a * b`  | Вираз      | Множення                                                                                                        |
| `T** pp` | Оголошення | Покажчик на покажчик на `T`                                                                                     |
| `**pp`   | Вираз      | Подвійне розіменування                                                                                          |

> Один і той самий символ у `C++` може мати різне значення залежно від контексту.
> Наприклад, `*` в оголошенні створює тип покажчика, а `*` у виразі отримує значення за адресою.

## Семантика покажчика

Для правильного розуміння покажчиків необхідно чітко розділяти три різні сутності:

- Сам об’єкт у пам’яті
- Адресу цього об’єкта
- Покажчик, який зберігає адресу

*Приклад:*

>```cpp
>int x = 10;
>int* p = &x;
>```

*У пам'яті:*

| Сутність | Значення                            |
|----------|-------------------------------------|
| `x`      | `10`                                |
| `&x`     | адреса змінної `x`                  |
| `p`      | адреса `x`                          |
| `*p`     | значення за адресою `p`, тобто `10` |


### Оператор взяття адреси &

Оператор `&` повертає адресу об’єкта в пам’яті.

*Приклад:*

> ```cpp
> int x = 5;
> cout << &x;
> ```

Тут буде виведено адресу змінної `x`.

Адреса це число, яке визначає місце розташування об’єкта в оперативній пам’яті.

*Приклад:*

> ```cpp
> int x = 5;
> int* p = &x;
> ```

*Схематично:*

> ```text
> x        -> 5
> &x       -> 0x1000
> p        -> 0x1000
> *p       -> 5
> ```


### Оператор розіменування *

Оператор `*` у виразі означає: «взяти значення за адресою».

*Приклад:*

> ```cpp
> int x = 10;
> int* p = &x;
> 
> cout << *p;
> ```

Результат `10`, тобто `*p` це фактично доступ до самої змінної `x`.

### Зміна значення через покажчик

Через покажчик можна змінювати значення об’єкта.

> ```cpp
> int x = 10;
> int* p = &x;
>
> *p = 25;
> ```

Після цього `x` буде мати значення `25`.
Оскільки `*p` це прямий доступ до області пам’яті, де зберігається `x`.

### Зміна адреси покажчика

Покажчик можна перенаправити на інший об’єкт.

> ```cpp
> int x = 10;
> int y = 20;
> 
> int* p = &x;
> 
> p = &y;
> ```

Тепер `*p == 20`, змінна `x` при цьому не змінюється.
Змінюється лише адреса, яка зберігається в `p`.


### Різниця між зміною адреси і зміною значення

| Операція   | Що змінюється               |
|------------|-----------------------------|
| `p = &y`   | Адреса всередині покажчика  |
| `*p = 100` | Значення об’єкта за адресою |
| `p++`      | Адреса покажчика            |
| `(*p)++`   | Значення об’єкта            |

### Пріоритет операцій * і ++

| Вираз    | Реальний порядок виконання | Що відбувається                                                          |
|----------|-----------|--------------------------------------------------------------------------|
| `*p++`   | `*(p++)`                   | Спочатку використовується поточна адреса, після чого покажчик зміщується |
| `*++p`   | `*(++p)`                   | Спочатку покажчик зміщується, потім виконується розіменування            |
| `(*p)++` | `(*p)++`                   | Збільшується значення за адресою                                         |
| `++(*p)` | `++(*p)`                   | Спочатку збільшується значення, потім повертається нове значення         |
| `++*p`   | `++(*p)`                   | Префіксний інкремент значення за адресою                                 |
| `*p--`   | `*(p--)`                   | Використовується поточна адреса, потім покажчик зменшується              |
| `*--p`   | `*(--p)`                   | Покажчик зміщується назад, потім виконується розіменування               |
| `(*p)--` | `(*p)--`                   | Значення за адресою зменшується                                          |
| `++p`    | `++p`                      | Зміщується сам покажчик                                                  |
| `p++`    | `p++`                      | Використовується поточна адреса, потім покажчик зміщується               |

*Приклад*

> ```cpp
> int A[] = {10, 20, 30};
> int* p = A;
> ```

| Вираз    | Результат | Новий стан `p`     |
|----------|-----------|--------------------|
| `*p++`   | `10`      | `p` вказує на `20` |
| `*++p`   | `30`      | `p` вказує на `30` |
| `(*p)++` | `30`      | `A[2]` стає `31`   |

### Нульовий покажчик

| Конструкція         | Значення                           | Особливість                      |
|---------------------|------------------------------------|----------------------------------|
| `nullptr`           | Спеціальне значення «немає адреси» | Сучасний стандарт C++            |
| `NULL`              | Макрос нульового значення          | Застарілий стиль із C            |
| `int* p = nullptr;` | Покажчик не вказує на об’єкт       | Безпечна ініціалізація           |
| `p == nullptr`      | Перевірка валідності               | Перед розіменуванням             |
| `p != nullptr`      | Покажчик валідний                  | Можна працювати з пам’яттю       |
| `*p` при `nullptr`  | Невизначена поведінка              | Може завершити програму аварійно |

### Висячий покажчик

| Ситуація                            | Що відбувається                         | Небезпека                |
|-------------------------------------|-----------------------------------------|--------------------------|
| `delete p`                          | Пам’ять звільнена                       | Адреса в `p` залишається |
| `free(p)`                           | Пам’ять повернута ОС                    | Покажчик стає невалідним |
| `*p` після `delete`                 | Доступ до неіснуючої пам’яті            | Undefined Behavior       |
| Повернення адреси локальної змінної | Адреса стекового об’єкта стає недійсною | Невалідний доступ        |
| Використання після звільнення       | Use After Free                          | Критична помилка безпеки |

*Небезпечний приклад:*

> ```cpp
> int* p = new int(10);
> 
> delete p;
> 
> cout << *p;
> ```

*Безпечний варіант:*

> ```cpp
> delete p;
> p = nullptr;
> ```

**Основні причини появи висячих покажчиків**

| Причина                      | Приклад               |
|------------------------------|-----------------------|
| Подвійне звільнення          | `delete p; delete p;` |
| Використання після `delete`  | `cout << *p;`         |
| Повернення локальної адреси  | `return &x;`          |
| Невалідний час життя об’єкта | Об’єкт уже знищений   |

### Константність і покажчики

**Покажчик на константу**

| Конструкція    | Значення                              |
|----------------|---------------------------------------|
| `const int* p` | Не можна змінювати значення через `p` |
| `int const* p` | Те саме                               |
| `p = &y`       | Дозволено                             |
| `*p = 5`       | Помилка                               |


*Приклад:*

> ```cpp
> int x = 10;
> int y = 20;
> 
> const int* p = &x;
> 
> p = &y;
> ```

**Константний покажчик**

| Конструкція         | Значення                |
|---------------------|-------------------------|
| `int* const p = &x` | Не можна змінити адресу |
| `p = &y`            | Помилка                 |
| `*p = 5`            | Дозволено               |

*Приклад:*

> ```cpp
> int x = 10;
> 
> int* const p = &x;
> 
> *p = 50;
> ```

**Константний покажчик на константу**

| Конструкція               | Значення                                  |
|---------------------------|-------------------------------------------|
| `const int* const p = &x` | Не можна змінювати ні адресу, ні значення |
| `p = &y`                  | Помилка                                   |
| `*p = 5`                  | Помилка                                   |


*Приклад:*

> ```cpp
> int x = 10;
> 
> const int* const p = &x;
> ```

**Порівняльна таблиця константності**

| Конструкція          | Зміна адреси | Зміна значення |
|----------------------|--------------|----------------|
| `int* p`             | Так          | Так            |
| `const int* p`       | Так          | Ні             |
| `int* const p`       | Ні           | Так            |
| `const int* const p` | Ні           | Ні             |

**Правило читання справа наліво**

| Конструкція          | Читання                                          |
|----------------------|--------------------------------------------------|
| `int* p`             | `p` це покажчик на `int`                         |
| `const int* p`       | `p` це покажчик на константний `int`             |
| `int* const p`       | `p` це константний покажчик на `int`             |
| `const int* const p` | `p` це константний покажчик на константний `int` |

### Сучасний C++ і покажчики

У сучасному C++ сирі покажчики T* використовуються значно рідше, ніж у класичному C.

- Причини:
  - Ризик витоків пам’яті
  - Висячі покажчики
  - Невизначена поведінка
  - Ручне керування життєвим циклом

**Тому у сучасному C++ є спеціальні засоби:**

| Засіб             | Призначення                     |
|-------------------|---------------------------------|
| `std::unique_ptr` | Єдиний власник ресурсу          |
| `std::shared_ptr` | Спільне володіння               |
| `std::weak_ptr`   | Слабке посилання                |
| `std::vector`     | Динамічний масив                |
| `std::array`      | Безпечний статичний масив       |
| `std::span`       | Представлення діапазону пам’яті |
| Посилання `T&`    | Безпечна альтернатива покажчику |

Сирі покажчики використовувались і зараз використовуються ближче до заліза

- У низькорівневому програмуванні
- У драйверах
- У ядрах ОС
- У вбудованих системах
- У `high performance` коді
- В `allocator` та `memory pool` системах
- У контейнерах стандартної бібліотеки
- У внутрішніх механізмах STL
- У взаємодії з C API
- У системному програмуванні Linux та Windows


## Арифметика покажчиків

> Арифметика покажчиків це операції над адресами пам’яті.
> Зміщення виконується не в байтах, а в елементах відповідного типу.

**Допустимі операції над покажчиками:**

| Операція                | Приклад    | Результат                        | Особливість            |
|-------------------------|------------|----------------------------------|------------------------|
| Додавання числа         | `p + 1`    | Перехід до наступного елемента   | Зсув на `sizeof(T)`    |
| Віднімання числа        | `p - 1`    | Перехід до попереднього елемента | Зсув назад             |
| Інкремент               | `p++`      | Перехід до наступного елемента   | Аналог `p = p + 1`     |
| Декремент               | `p--`      | Перехід назад                    | Аналог `p = p - 1`     |
| Різниця покажчиків      | `p2 - p1`  | Кількість елементів між адресами | Лише для одного масиву |
| Порівняння              | `p1 == p2` | Порівняння адрес                 | Не значень             |
| Розіменування зі зсувом | `*(p + i)` | Доступ до елемента               | Еквівалент `p[i]`      |

*Приклад адресної арифметики:*

> ```cpp
> int main()
> {
>     int A[] = {10, 20, 30, 40};
> 
>     int* p = A;
> 
>     std::cout << *p << '\n';
>     std::cout << *(p + 1) << '\n';
>     std::cout << *(p + 2) << '\n';
> 
>     p++;
> 
>     std::cout << *p << '\n';
> }
> ```

*Результат*

> ```text
> 10
> 20
> 30
> 20
> ```

*Особливості:*

| Вираз   | Еквівалент |
|---------|------------|
| `A[i]`  | `*(A + i)` |
| `p[i]`  | `*(p + i)` |
| `&A[i]` | `A + i`    |

- Зсув в пам'яті виконується на кількість байтів, що відповідає типу, на який вказує покажчик.
- Операції з покажчиками можуть виконуватися тільки з покажчиками на однакові типи.
- Покажчики на різні типи не можуть бути порівняні або віднімані один від одного.

## Покажчик типу void*

> `void*` це узагальнений покажчик, який може зберігати адресу об’єкта будь якого типу.

**Основні властивості `void*`**

| Властивість                                        | Опис                               |
|----------------------------------------------------|------------------------------------|
| Не має конкретного типу                            | Тип об’єкта невідомий              |
| Може зберігати будь яку адресу                     | `int`, `double`, `char`, структура |
| Не можна розіменувати напряму                      | Потрібне приведення типу           |
| Активно використовується в C API                   | Універсальні інтерфейси            |
| Не підтримує адресну арифметику в стандартному C++ | Розмір типу невідомий              |

*Приклад:*

> ```cpp
> int main()
> {
> int x = 10;
> 
>     void* p = &x;
> 
>     std::cout << *(static_cast<int*>(p));
> }
> ```

Приведення типу потрібне, тому що компілятор не знає:

- який розмір об’єкта
- як інтерпретувати байти
- як виконувати розіменування

Тому `*(static_cast<int*>(p))` спочатку перетворює `void*` у `int*`.


## Одновимірні масиви і покажчики

> Ім’я масиву це адреса першого елемента масиву.

> ```cpp
> A == &A[0]
> ```

*Еквівалентні записи*

| Запис      | Значення        |
|------------|-----------------|
| `A[i]`     | Елемент масиву  |
| `*(A + i)` | Те саме         |
| `p[i]`     | Те саме         |
| `*(p + i)` | Те саме         |
| `&A[i]`    | Адреса елемента |
| `A + i`    | Те саме         |

*Приклад проходження масиву через покажчик:*

>```cpp
> int main()
> {
>     int A[] = {5, 10, 15, 20};
> 
>     int* p = A;
> 
>     for (int i = 0; i < 4; i++)
>         std::cout << *(p + i) << ' ';
> }
> ```

*Результат:*

```text
5 10 15 20
```

*Схема пам’яті:*

> ```text
> A[0]  A[1]  A[2]  A[3]
>  5     10    15    20
> 
>  ^
>  p
> ```

## Двовимірні масиви і покажчики

Двовимірний масив у C++ зберігається послідовно по рядках `int A[2][3];`.

*Структура пам’яті:*

> ```text
> A[0][0] A[0][1] A[0][2] A[1][0] A[1][1] A[1][2]
> ```

**Доступ до елементів**

| Запис             | Значення                       |
|-------------------|--------------------------------|
| `A[i][j]`         | Звичайний доступ               |
| `*(*(A + i) + j)` | Доступ через покажчики         |
| `*(A[i] + j)`     | Доступ через покажчик на рядок |


*Приклад:*

> ```cpp
> int main()
> {
>     int A[2][3] =
>     {
>         {1, 2, 3},
>         {4, 5, 6}
>     };
> 
>     std::cout << A[1][2] << '\n';
> 
>     std::cout << *(*(A + 1) + 2);
> }
>```

*Результат:*

> ```text
> 6
> 6
> ```

## Виділення і звільнення пам’яті

**Порівняльна таблиця C і C++**

| Операція                 | C        | C++        |
|--------------------------|----------|------------|
| Виділення одного об’єкта | `malloc` | `new`      |
| Звільнення               | `free`   | `delete`   |
| Виділення масиву         | `malloc` | `new[]`    |
| Звільнення масиву        | `free`   | `delete[]` |
| Виклик конструктора      | Ні       | Так        |
| Типобезпечність          | Низька   | Вища       |

*Приклад `new/delete`:*

> ```cpp
> int main()
> {
>     int* p = new int(10);
> 
>     std::cout << *p << '\n';
> 
>     delete p;
> }
>```

*Приклад `new[]/delete[]`:*

> ```cpp
> int main()
> {
>     int size = 5;
> 
>     int* A = new int[size];
> 
>     for (int i = 0; i < size; i++)
>     {
>         A[i] = i * 10;
>     }
> 
>     for (int i = 0; i < size; i++)
>     {
>         std::cout << A[i] << ' ';
>     }
> 
>     delete[] A;
> }
>```

## Розміщення одновимірних і двовимірних масивів у пам’яті

| Масив          | Розміщення у пам’яті                              |
|----------------|---------------------------------------------------|
| `int A[5];`    | `A[0] A[1] A[2] A[3] A[4]`                        |
| `int B[2][3];` | `B[0][0] B[0][1] B[0][2] B[1][0] B[1][1] B[1][2]` |

*У двовимірних масивах пам’ять також безперервна, але елементи групуються по рядках.*

**Порівняння:**

| Характеристика     | Одновимірний масив | Двовимірний масив    |
|--------------------|--------------------|----------------------|
| Кількість індексів | Один               | Два                  |
| Структура          | Лінійна            | Таблична             |
| Розміщення         | Послідовне         | Послідовне по рядках |
| Доступ             | `A[i]`             | `A[i][j]`            |
| Адреса елемента    | `A + i`            | `*(A + i) + j`       |
