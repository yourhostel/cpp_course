# ДИНАМІЧНІ СТРУКТУРИ ДАНИХ

## Мета

Оволодіння практичними навичками роботи з динамічними структурами даних 
на прикладі реалізації двозв’язного списку з ручним керуванням пам’яттю, 
перевіркою інваріантів та обробленням виняткових ситуацій.

## Вимоги

- Реалізувати власну структуру двозв’язного списку без використання STL
- Забезпечити операції додавання, доступу, зміни та видалення елементів
- Реалізувати перевірки коректності індексів і стану списку
- Використати винятки для оброблення помилок
- Продемонструвати роботу на користувацькому типі даних
- Перевантажити оператор порівняння для типу даних

## Реалізація

Реалізовано шаблонний контейнер `DoublyLinkedList<T>`, який зберігає елементи у вигляді двозв’язного списку.  
Кожен вузол містить дані та посилання на попередній і наступний вузли.

Для демонстрації використано структуру `Person`, де перевантажено `operator==` для порівняння за `id`.

### Таблиця методів

| Метод               | Опис                                         |
|---------------------|----------------------------------------------|
| `add(index, value)` | Вставляє елемент за індексом                 |
| `addFirst(value)`   | Додає елемент на початок                     |
| `addLast(value)`    | Додає елемент у кінець                       |
| `get(index)`        | Повертає елемент за індексом                 |
| `getFirst()`        | Повертає перший елемент                      |
| `getLast()`         | Повертає останній елемент                    |
| `set(index, value)` | Змінює значення за індексом                  |
| `remove(index)`     | Видаляє елемент за індексом                  |
| `remove(value)`     | Видаляє перший елемент, що дорівнює значенню |
| `removeFirst()`     | Видаляє перший елемент                       |
| `removeLast()`      | Видаляє останній елемент                     |
| `size()`            | Повертає кількість елементів                 |
| `isEmpty()`         | Перевіряє чи список порожній                 |
| `clear()`           | Очищує список                                |
| `copyFrom()`        | Копіює елементи з іншого списку              |
| `moveFrom()`        | Переміщує ресурси з іншого списку            |


## Інваріанти структури

У реалізації двозв’язного списку підтримуються такі інваріанти.

- Порожній список:
   `head_ == nullptr`, `tail_ == nullptr`, `size_ == 0` 
   забезпечується конструктором і `clear()`

- Голова списку:
   `head_->prev == nullptr`  
   забезпечується в `addFirst()` і `removeFirst()`

- Хвіст списку:
   `tail_->next == nullptr`  
   забезпечується в `addLast()` і `removeLast()`

- Узгодженість зв’язків:
   `prev->next` і `next->prev` завжди синхронні  
   забезпечується в `add(index)` і `remove(index)`

- Граничні випадки:
   при одному елементі `head_ == tail_`  
   при видаленні останнього елемента список стає порожнім

- Розмір:
   `size_` відповідає кількості вузлів  
   оновлюється у всіх `add/remove` і `clear()`

## Оброблення виняткових ситуацій

- Порожній список:
  при спробі доступу або видалення елементів з порожнього списку
  генерується `std::logic_error`  
  використовується в `removeFirst()`, `removeLast()`, `getFirst()`, `getLast()`

- Некоректний індекс:
  при виході індексу за межі генерується `std::out_of_range`  
  використовується в `checkIndex()` і `checkIndexForAdd()`

- Доступ до неіснуючого елемента:
  перевірка індексу виконується централізовано в `getNode()` 
  методи `get()`, `set()`, `remove(index)` покладаються на `getNode()`,  
  який генерує `std::out_of_range` при помилці

## Керування пам’яттю

Виділення пам’яті відбувається через `new` при створенні вузлів.  
Звільнення пам’яті виконується через `delete` в `remove...()`, `clear()` і деструкторі.
